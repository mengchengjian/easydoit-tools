# Redesign Skill

## How This Works

When applied to an existing project, follow this sequence:

1. **Scan** — Read the codebase. Identify the framework, styling method, and current design patterns.
2. **Diagnose** — Run through the audit below. List every generic pattern, weak point, and missing state you find.
3. **Fix** — Apply targeted upgrades working with the existing stack. Do not rewrite from scratch.

## Core Anti-Slop Checklist

- Typography: No Inter default, no missing weights, no orphaned words
- Color: No #000000 backgrounds, no oversaturated accents, one accent max
- Layout: No centered-everything, no 3-equal-cards, no h-screen
- Interactivity: Always full lifecycle states (loading, empty, error, hover, active, focus)
- Content: No lorem ipsum, no AI cliches (Elevate, Seamless, Unleash)
- Code: Semantic HTML, no div soup, GPU-accelerated animations
